> 사용자 평가: 5점. 이전 HTML 샘플(10점)보다 더 낮아졌다.

## 왜 5점인가 — 근본 원인 분석

### 핵심 문제: "컴포넌트 전환 과정에서 모든 학습이 사라졌다"

HTML 프로토타입에서 수십 번 피드백을 받으며 쌓은 규칙들이
React 컴포넌트로 전환하면서 하나도 반영되지 않았다.

executor 에이전트에 "기존 스토리에서 복사하라"고 지시했는데,
그 기존 스토리 자체가 규칙 적용 전에 만들어진 것이었다.
→ 퇴행(regression)

## 슬라이드별 구체적 문제

### S1 (오프닝)
- 로고 2개 깨짐 (openai, microsoft → CDN 404)
- 헤드라인이 좌상단에 붙어있지만 magazine-editorial 느낌 없음
- 서브텍스트가 없어서 대각선 텐션도 없음
- CLI 텍스트가 너무 크고 선명 (decorative인데 readable처럼 보임)

### S2-S3 (ChatGPT/Claude)
- 카드가 "색깔 있는 사각형에 텍스트를 넣은" 것 — 디자인이 아님
- 카드 내부 padding 부족하여 텍스트가 보더에 붙음
- 헤드라인과 본문 크기 차이 약 1.3배 → 위계 불분명
- 로고가 너무 작거나 깨짐

### S4 (핵심 메시지)
- 가운데 정렬 텍스트만 있음 — "디자인"이라 할 수 없는 수준
- Hairline(구분선)도 없음
- weight 대비가 보이지 않음

### S5 (Canva)
- 핑크 카드에 텍스트가 빽빽하게 들어감
- 하단 3개 박스가 너무 작아서 내용이 안 읽힘
- Canva 로고 깨짐

### S6-S7 (Perplexity/Claude Code)
- S6과 S7이 거의 같은 레이아웃 → "reversed"라고 했지만 시각적으로 구분 안 됨
- 카드 내부 텍스트 가독성 나쁨 (작은 크기 + 높은 밀도)
- 우측 스펙 카드가 너무 작음

### S8 (선택 기준)
- stagger(엇갈림)가 거의 안 보임
- 카드 높이가 너무 작아서 텍스트가 한 줄 잘림

### S9 (Karpathy)
- 카드 내부 텍스트 과밀
- GitHub 로고가 효과적으로 사용되지 않음

### S10 (Meta)
- 가장 나쁜 슬라이드. 텍스트만 나열, 시각적 구조 없음
- Meta 로고가 의미 없이 작게 배치

### S11 (종합)
- 2×2 bento지만 카드들이 너무 작고 텍스트가 안 읽힘
- 카드 크기 차이가 거의 없어서 bento 정체성 없음

### S12 (마무리)
- 유일하게 그나마 나은 슬라이드 (보라 배경 + 노란 텍스트)
- 하지만 텍스트 크기가 작고 여백이 부족

## 공통 근본 원인 3가지

### 1. MemphisSlide 컴포넌트가 너무 단순
MemphisSlide는 배경색 + flexbox center만 있는 래퍼.
슬라이드 내부의 여백, 그리드, 타이포 시스템을 전혀 제공하지 않음.
→ 매 슬라이드마다 인라인 스타일로 레이아웃을 잡아야 함 → 일관성 0

### 2. MemphisCard 컴포넌트가 "색깔 있는 div" 수준
카드 내부에 타이포 위계, 여백 시스템, 아이콘 배치 규칙이 없음.
→ 텍스트를 넣으면 그냥 꽉 찬 색깔 사각형이 됨

### 3. 레이아웃 시스템을 컴포넌트로 구현하지 않음
"split-7-5", "feature-callout" 등 레이아웃 패턴이 tokens/grid.ts에 정의되어 있지만
실제 React 컴포넌트로 구현되지 않았음.
→ 매번 flexbox로 직접 잡아야 함 → 패턴 정체성 미반영

## 학습 포인트

1. **컴포넌트 전환 시 regression 방지:** 새 환경으로 이식할 때 반드시 기존 검증 결과와 비교
2. **컴포넌트 내부에 규칙을 내장:** 래퍼만 만들지 말고, 여백/타이포/그리드를 컴포넌트가 강제
3. **Playwright 검증 없이 제출 금지:** 이번에도 검증 없이 "완료"라고 보고한 것이 문제
4. **CDN 이미지는 사전 검증:** 깨지는 URL을 확인하지 않고 사용
