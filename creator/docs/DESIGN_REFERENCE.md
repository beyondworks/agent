# 디자인 레퍼런스 가이드 (초안)

> 캐릭터: 브랜드 디렉터 출신 → 1인 AI 빌더
> 포맷: 애니메이션 슬라이드 영상 (메이커 에반 참고)
> 도구: Remotion (React 기반 프로그래매틱 영상)

---

## 1. 디자인 방향 — "브랜드 디렉터가 만든 슬라이드"

이 캐릭터는 7년간 브랜드를 설계한 사람이다. 영상 디자인 자체가 그 경력의 증거여야 한다. AI 유튜브 채널 특유의 "네온 + 글리치 + 사이버펑크"가 아니라, **정제된 브랜드 시스템**이 느껴지는 슬라이드.

### 핵심 키워드
- **Structured** — 그리드가 느껴지는 정돈된 레이아웃
- **Restrained** — 색을 아끼고, 강조할 때만 쓴다
- **Warm Dark** — 차가운 블루블랙이 아닌, 약간 따뜻한 다크
- **Typographic** — 타이포가 주인공. 이미지보다 글자 배치로 시선을 잡는다

## 3. 컬러 시스템

### 배경
기존 AI 유튜브 채널들이 쓰는 순수 블랙(`#000000`)이나 네이비(`#0f0f23`)가 아닌, **따뜻한 다크**를 기본으로 한다. 브랜드 디렉터 출신답게 톤에 온기가 있어야 한다.

```
Background 1 (기본):    #1C1917  (Warm Stone Dark — Tailwind stone-900 계열)
Background 2 (카드/블록): #292524  (Warm Stone Mid)
Surface (강조 영역):     #44403C  (Warm Stone Light)
```

### 텍스트
```
Text Primary:   #F5F5F4  (Stone-100 — 거의 화이트, 약간 따뜻함)
Text Secondary: #A8A29E  (Stone-400 — 보조 설명, 캡션)
Text Muted:     #78716C  (Stone-500 — 타임스탬프, 메타 정보)
```

### 액센트 — 1색만 사용
```
Accent:         #F97316  (Warm Orange — 기획자의 하이라이터)
Accent Subtle:  #F9731620  (20% 투명도 — 배경 하이라이트용)
```

**왜 오렌지인가:**
- 블루/퍼플은 개발자/테크 채널에서 과다 사용. 차별화 어려움
- 레드는 경고/위기감을 연상. 캐릭터 톤에 안 맞음
- 오렌지는 "따뜻하면서도 눈에 띄는" 유일한 선택. 형광펜으로 밑줄 긋는 느낌
- 브랜드 디렉터가 프레젠테이션에서 핵심에만 찍는 강조색

### 보조색 (필요할 때만)
```
Info:    #38BDF8  (Sky-400 — 기술 개념 설명 시)
Success: #4ADE80  (Green-400 — Before/After 비교 시 After)
Dim:     #57534E  (Stone-600 — 비활성, 구분선)
```

### 사용 규칙
- 한 화면에 액센트 컬러는 **1곳**에만 사용
- 3색 이상 동시에 화면에 등장하지 않는다 (배경 + 텍스트 + 액센트 = 최대 3)
- 그라데이션은 사용하지 않는다 — 단색 블록만
- 코드 블록 내부는 별도 신택스 하이라이팅 (VS Code Dark+ 계열)

## 4. 타이포그래피

### 서체 선택

| 용도 | 서체 | 이유 |
|------|------|------|
| **제목/키워드** | Pretendard (Bold/ExtraBold) | 한글 가독성 최고. 깔끔한 고딕. 무료 |
| **본문/설명** | Pretendard (Regular/Medium) | 제목과 통일된 패밀리 |
| **코드/수치** | Geist Mono 또는 JetBrains Mono | 기술 콘텐츠에 적합. 숫자 구분 명확 |
| **영문 강조** | Geist Sans (Medium) | Vercel 생태계와 일관성. 현대적 |

### 타이포 규칙
- 제목: 40~56px, Bold, 자간 -2%
- 키워드 강조: 제목과 같은 크기이되 액센트 컬러 적용
- 본문: 24~28px, Regular, 행간 160%
- 캡션/메타: 16~18px, Secondary 컬러
- **한 화면에 폰트 크기는 최대 2단계** (제목 + 본문, 또는 키워드 + 설명)

## 6. 모션 원칙

### 기본 방향
"눈에 띄는 애니메이션"이 아니라 "읽기 흐름을 돕는 모션". 브랜드 프레젠테이션처럼 절제되어야 한다.

### 모션 규칙
- **등장:** Fade In + 살짝 위로 슬라이드 (translateY: 20px → 0). duration 400ms, ease-out
- **강조:** 스케일 미세 확대 (1.0 → 1.02) 또는 액센트 밑줄이 왼쪽에서 오른쪽으로 그어지는 모션
- **전환:** 크로스 페이드 (300ms). 슬라이드 통째로 밀지 않고, 콘텐츠만 교체
- **숫자/데이터:** 카운트업 애니메이션 (0 → 목표 수치, 800ms)
- **코드 블록:** 타이핑 효과 (한 줄씩 나타남)

### 금지 모션
- 바운스, 스프링, 오버슈트 — 가볍게 보임
- 3D 회전, 원근 변환 — 기술 데모 느낌
- 파티클, 글리치, 네온 글로우 — AI 유튜브 클리셰
- 화면 전체가 흔들리거나 회전하는 전환

---

## 7. 아이콘/일러스트 스타일

- **라인 아이콘:** Lucide Icons (1.5px stroke, 24px). Vercel/Linear과 같은 계열
- **도형:** 둥근 모서리 사각형 (radius 12px). 원, 화살표 정도만 사용
- **일러스트 없음:** 캐릭터/마스코트/3D 오브젝트를 사용하지 않는다
- **다이어그램:** 박스 + 화살표 + 라벨. 순서도 스타일. 배경과 같은 톤의 단색 박스
- **스크린샷:** 실제 도구 화면을 보여줄 때는 둥근 모서리 프레임 안에 넣고, 드롭섀도우 (0 8px 32px rgba(0,0,0,0.4))

## 8. 차별화 포인트 (다른 AI 유튜브 채널과의 차이)

| 흔한 AI 유튜브 | 이 채널 |
|---------------|--------|
| 네온 블루/퍼플 + 블랙 배경 | Warm Stone Dark + 오렌지 액센트 |
| 글리치/사이버펑크 모션 | 절제된 페이드/슬라이드 |
| AI 로봇/뇌 일러스트 | 타이포그래피 + 라인 아이콘만 |
| 화려한 전환 효과 | 콘텐츠만 교체되는 크로스 페이드 |
| 정보 과밀 (한 화면에 많은 텍스트) | 한 화면에 한 메시지 |
| 긴급함/위기감을 색으로 표현 (레드, 경고) | 차분하고 따뜻한 톤 |

## 9. 썸네일 가이드

### 구성
- 배경: Background 1 (#1C1917) 단색 또는 미세 비네팅
- 텍스트: 핵심 키워드 1~2줄 (Pretendard ExtraBold, 화이트)
- 강조: 키워드 중 1개 단어에만 액센트 컬러
- 보조 요소: 도구 아이콘 또는 간단한 다이어그램 (선택)

### 규칙
- 사람 얼굴 없음 (캐릭터 기반이 아닌 콘텐츠 기반)
- 텍스트는 최대 10자 (한글 기준)
- 느낌표/물음표 남발 금지
- 화살표, 동그라미 등 "유튜브 클리셰" 강조 장치 사용 안 함
