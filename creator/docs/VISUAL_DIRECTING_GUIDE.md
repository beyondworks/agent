# 비주얼 디렉팅 학습 가이드

> Playwright 캡처를 통해 분석한 디자인 출력물의 문제점과 개선 규칙.
> AI(Claude)가 디자인 작업 시 반드시 참조해야 하는 자기 교정 문서.

---

## 1. 캡처 분석 — Memphis × Biotech 샘플

### 잘 된 점
- 카드별 다른 색상의 오프셋 섀도우 → Memphis 특성 반영
- 인용 슬라이드의 노란색 하이라이트 마킹 → 스타일 고유 장치
- VS 뱃지의 각진 형태 + 굵은 보더 → 스타일 일관성
- 코드 블록의 bad/good 구분 → 컬러로 직관적 구분

### 문제점 (자기 진단)

| # | 문제 | 원인 | 개선 규칙 |
|---|------|------|----------|
| 1 | **장식 요소가 너무 작고 희미함** | opacity를 0.15~0.25로 낮게 잡음. 삼각형/원이 거의 안 보임 | Memphis 장식은 **최소 opacity 0.3**, 크기는 슬라이드 대비 **8~15%** |
| 2 | **텍스트 크기 계층이 불명확** | h1, h2, body 간 크기 차이가 작음 | 제목:본문 비율 최소 **2.5:1** (예: h1=48px → body=18px) |
| 3 | **여백이 균일해서 리듬감이 없음** | 모든 패딩을 비슷한 %로 설정 | 요소 간 간격에 **강약** 넣기: 제목↔본문 = 좁게, 섹션↔섹션 = 넓게 |
| 4 | **카드 내부 패딩이 부족** | padding 16-20px → 콘텐츠가 보더에 붙어 보임 | 카드 내부 패딩은 **보더 두께의 6~8배** (3px 보더 → 18~24px 패딩) |
| 5 | **Storybook 쇼케이스의 컴포넌트가 정적** | 단순 나열, 시각적 계층 없음 | 쇼케이스에도 **시각적 흐름** 적용: 큰 요소 → 중간 → 작은 순서 |
| 6 | **라이트 배경에서 콘텐츠 영역 구분이 약함** | 배경과 카드 배경 차이가 작음 | 라이트 모드에서 카드 배경은 **순수 화이트(#FFF)**, 슬라이드 배경은 **약간 톤 있는 색** |

## 2. 스타일별 비주얼 디렉팅 규칙

### Memphis Design 적용 시
- 보더 두께: **3~4px** (절대 1~2px 아님)
- 오프셋 섀도우: **5~8px** (보더 두께의 1.5~2배)
- border-radius: **0** (각진 게 핵심)
- 장식 요소: **슬라이드당 2~3개**, opacity **0.2~0.4**
- 색상 변주: 카드마다 **다른 액센트 섀도우** (핑크, 옐로우, 틸 순환)
- 타이포: heading은 **Black(900)**, body는 **Regular(400)** — 무게 대비 극대화

### Glassmorphism 적용 시
- blur: **16~24px** (너무 낮으면 유리 효과 안 보임)
- 배경 opacity: **0.03~0.08** (너무 높으면 불투명하게 보임)
- 보더: **rgba(255,255,255,0.06~0.1)** — 미세하지만 존재해야 함
- 글로우: accent 색상의 **0.1~0.15 opacity**
- 배경에 **반드시 비비드한 요소**(orb, 그라데이션)가 있어야 블러가 돋보임

### Neobrutalism 적용 시
- 보더: **3px solid #000** (검정 보더가 핵심)
- 오프셋 섀도우: **4~6px #000** (보더와 같은 색)
- border-radius: **8~12px** (Brutalism과 다르게 둥근 모서리)
- 배경색: **밝고 비비드한 단색** (파스텔 아님)
- 호버: 섀도우 방향 반전 또는 없어짐

## 3. 공통 비주얼 규칙 (모든 스타일)

### 타이포그래피 계층
```
Display (슬라이드 제목):  본문의 3~4배
H1 (섹션 제목):          본문의 2.5~3배
H2 (카드 제목):          본문의 1.8~2배
Body (본문):             기준 (1x)
Caption (캡션):          본문의 0.75배
Label (라벨/태그):       본문의 0.6배
```

한글 적용 시: **행간(line-height)은 영문보다 10~15% 넓게** (영문 1.5 → 한글 1.65~1.75)

### 여백 리듬
```
요소 내부:     패딩은 보더 두께의 6~8배
요소 간 (같은 그룹):   기본 간격의 1x (예: 12px)
그룹 간:              기본 간격의 2~3x (예: 24~36px)
섹션 간:              기본 간격의 4~6x (예: 48~72px)
슬라이드 가장자리:     콘텐츠 영역의 8~12%
```

### 색상 사용
- **한 슬라이드에 3색 이내** (배경 + 텍스트 + 액센트)
- 강조색은 **화면의 10~15%만** 차지
- 비교 슬라이드에서 Before는 **동일 색상의 낮은 opacity**, After는 **원래 강도**

### 장식 요소
- **슬라이드당 최대 3개** (넘으면 산만)
- 콘텐츠 영역과 **겹치지 않게** (가장자리에 배치)
- 크기는 슬라이드 면적의 **5~15%**
- 장식은 **스타일의 정체성을 보여주는 요소**여야 함 (의미 없는 장식 금지)

## 4. Storybook DNA 활용 규칙

### 새 조합을 만들 때의 체크리스트

```
□ Style Composer에서 해당 조합을 먼저 확인했는가?
□ 팔레트의 isDark 속성에 맞게 배경/텍스트를 설정했는가?
□ 스타일 프리셋의 구조적 특성(radius, shadow, border)을 정확히 적용했는가?
□ 폰트의 importUrl로 실제 폰트를 로드했는가?
□ 한글 행간을 영문보다 10~15% 넓게 설정했는가?
□ 타이포 계층 비율(3:2.5:1.8:1:0.75:0.6)을 지켰는가?
□ 여백 리듬(내부 > 요소간 > 그룹간 > 섹션간)을 적용했는가?
□ 한 슬라이드에 3색 이내인가?
□ 장식 요소가 3개 이하이고, 콘텐츠와 겹치지 않는가?
□ Playwright로 캡처하여 시각적으로 확인했는가?
```

### 스타일 적용 우선순위
1. **구조** (radius, shadow 패턴) → 스타일이 결정
2. **색상** → 팔레트가 결정
3. **타이포** → 폰트셋이 결정
4. **장식** → 스타일 + 팔레트의 조합으로 결정
5. **여백** → 공통 규칙 적용 (스타일 무관)

## 5. 자기 교정 프로세스

```
1. 코드 작성
2. Playwright로 캡처
3. 이 문서의 체크리스트로 검증
4. 문제 발견 시: 이 문서의 규칙 참조하여 수정
5. 재캡처 → 재검증
6. 통과 후 사용자에게 제출
```

**핵심: 코드만 보고 "됐다"고 하지 않는다. 반드시 시각적 캡처로 확인한다.**
