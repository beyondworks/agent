# 비주얼 감사 — Exaggerated Minimalism × Government 샘플

> Playwright 캡처 기반 자가 진단. 규칙 대비 문제점 분석.

---

## 슬라이드별 분석

### S1 — Title (magazine-editorial)

**잘 된 점:**
- 헤드라인 크기가 충분히 크고 weight 900 적용됨
- 액센트 컬러(블루)가 단일색으로 잘 제한됨
- CLI 텍스트가 장식으로 적절히 흐림

**문제점:**
| # | 문제 | 위반 규칙 | 심각도 |
|---|------|---------|-------|
| 1 | 헤드라인이 수직 중앙보다 위에 붙어있음 — 상단 여백 부족 | verticalPlacementRules: 광학적 중앙 | HIGH |
| 2 | 서브텍스트와 헤드라인 사이 간격이 너무 넓음 — 관계가 분리되어 보임 | spacingRhythm: 같은 그룹 요소는 가깝게 | MEDIUM |
| 3 | CLI 텍스트가 하단에 고립되어 빈 공간이 어색함 | 여백의 분포가 불균일 | LOW |
| 4 | magazine-editorial 패턴인데 에디토리얼 특유의 비대칭/드라마틱한 타이포 배치가 아님. 그냥 좌정렬 | layoutVariations: 패턴의 정체성 미반영 | HIGH |

### S2 — Problem (asymmetric-8-4)

**잘 된 점:**
- 8:4 비대칭 분할이 적용됨
- 우측 번호 항목에 헤어라인 구분 적절
- 번호(01/02/03)에 블루 액센트 적용

**문제점:**
| # | 문제 | 위반 규칙 | 심각도 |
|---|------|---------|-------|
| 1 | 좌측 헤드라인이 너무 크고 검정이라 우측 항목과의 시각적 무게 균형이 안 맞음 | hierarchy: 50/30/15/5인데 좌측이 90%를 차지 | HIGH |
| 2 | 우측 항목들이 슬라이드 중앙보다 위에 몰려있고 하단이 비어있음 | verticalPlacement: 광학적 중앙 보정 필요 | MEDIUM |
| 3 | 본문 텍스트("아이디어와 전략은...")가 2줄로 깨지는데 1줄이면 충분 | lineBreakRules: 불필요한 줄바꿈 | LOW |

### S3 — Quote (quote-spotlight)

**잘 된 점:**
- 극단적 여백 확보
- 헤어라인 위아래 배치
- "설계다"가 블루 + weight 대비로 강조

**문제점:**
| # | 문제 | 위반 규칙 | 심각도 |
|---|------|---------|-------|
| 1 | "AI 시대의 핵심 역량은 코딩이 아니라"가 2줄로 깨짐 — "코딩이 아니라"가 따로 떨어져서 리듬 깨짐 | lineBreakRules: 의미 단위 줄바꿈, 자동 줄바꿈 지양 | HIGH |
| 2 | "설계다"만 단독 줄인데, 위의 2줄 + 아래 1줄 = 비대칭이 의도적으로 보이지 않음 | typography hierarchy: 의도된 비대칭이어야 함 | MEDIUM |
| 3 | 출처 텍스트가 너무 작아서 거의 안 보임 | decorative vs readable 경계 모호 | LOW |

### S4 — Compare (alternating-zigzag)

**잘 된 점:**
- zigzag 패턴 시도 (상단 좌정렬, 하단 우정렬)
- 화살표 흐름이 프로세스를 시각화

**문제점:**
| # | 문제 | 위반 규칙 | 심각도 |
|---|------|---------|-------|
| 1 | "기존 방식" 행과 "AI + 설계 역량" 행의 시각적 무게가 동일함 — Before는 약하고 After는 강해야 하는데 구분이 안 됨 | colorContrastRules: Before=muted, After=primary+accent | HIGH |
| 2 | 화살표(→)들이 너무 작고 약해서 흐름이 잘 안 읽힘 | readability: 흐름 표시자가 충분히 눈에 띄어야 함 | HIGH |
| 3 | 상단 행이 헤어라인 위에, 하단 행이 아래에 있는데 두 행 사이 간격이 너무 넓음 — 비교 관계가 시각적으로 연결 안 됨 | spacingRhythm: 비교 대상은 가깝게 | MEDIUM |
| 4 | 전체 콘텐츠가 수직 중앙보다 위에 몰림 | verticalPlacement | MEDIUM |

### S5 — List (feature-callout)

**잘 된 점:**
- 8:4 비대칭으로 주인공(01)이 크게 표현됨
- 02/03이 우측에 스택으로 종속적 위치

**문제점:**
| # | 문제 | 위반 규칙 | 심각도 |
|---|------|---------|-------|
| 1 | 좌측 "01 구조 설계"가 상단에 몰려있고 하단 50%가 비어있음 | verticalPlacement: 광학적 중앙 | HIGH |
| 2 | 좌측 본문이 2줄인데 우측 02/03 본문도 각 1줄 → 정보 밀도 차이가 너무 큼 | hierarchy: 주인공과 보조의 정보량 균형 | MEDIUM |
| 3 | 02와 03 사이 헤어라인이 있는데, 01과 02/03 사이는 구분이 없어서 그룹핑 관계가 불명확 | 시각적 그룹핑 일관성 | MEDIUM |

### S6 — Code (staggered-rows)

**잘 된 점:**
- staggered(엇갈림) 배치 시도
- "나쁜 브리프"는 짧고, "좋은 브리프"는 길어서 콘텐츠 대비 효과

**문제점:**
| # | 문제 | 위반 규칙 | 심각도 |
|---|------|---------|-------|
| 1 | 헤드라인 "좋은 브리프 vs 나쁜 브리프"가 너무 크고 진함 — 코드 블록보다 헤드라인이 압도적 | hierarchy: 이 슬라이드의 주인공은 코드 블록이어야 함 | HIGH |
| 2 | "나쁜 브리프" 블록이 너무 작음 (콘텐츠가 1줄) — "좋은 브리프" 블록과 시각적 균형 안 맞음 | containerRules: 콘텐츠에 맞되 비교 시 비슷한 존재감 필요 | HIGH |
| 3 | 두 블록의 보더가 거의 동일해서 "나쁜/좋은" 구분이 컬러 레이블에만 의존 | colorContrastRules: 배경색 차이로 즉시 구분 가능해야 함 | MEDIUM |
| 4 | "좋은 브리프" 텍스트 마지막 줄이 줄바꿈되어 잘림 | lineBreakRules: 컨테이너 크기 조정 | LOW |

### S7 — CTA (full-bleed)

**잘 된 점:**
- 다크 네이비 풀블리드로 다른 슬라이드와 확실한 전환
- 텍스트가 중앙 정렬, 심플

**문제점:**
| # | 문제 | 위반 규칙 | 심각도 |
|---|------|---------|-------|
| 1 | "가장 잘 만듭니다"에 weight 대비가 있는데 블루 등 액센트 컬러가 없음 — 미니멀이어도 강조점은 필요 | emphasis: 키워드에 accent 적용 | MEDIUM |
| 2 | 서브텍스트가 너무 길어서 1줄인데도 빼곡함 | lineBreakRules: 문장을 줄이는 것도 방법 | LOW |

## 공통 패턴 (반복되는 문제)

### 1. 수직 배치 일관성 부족 (5/7 슬라이드에서 발생)
- S1, S2, S4, S5에서 콘텐츠가 상단에 몰림
- **근본 원인:** `justify-content: center`가 적용되어도 콘텐츠 높이가 작으면 상단 padding 영역에 몰리는 것처럼 보임
- **해결:** 콘텐츠 영역 자체를 슬라이드 중앙에 명시적으로 배치 (margin: auto)

### 2. "스타일의 정체성"이 레이아웃에 반영 안 됨
- Exaggerated Minimalism은 "극단적 여백"이 정체성인데, 실제로는 60~65% 정도 여백
- **해결:** 콘텐츠 영역을 슬라이드의 40% 이내로 제한해야 진짜 "exaggerated" 느낌

### 3. Before/After, Bad/Good 비교에서 시각적 무게 차이 부족
- S4(Compare)와 S6(Code)에서 "나쁜 것"과 "좋은 것"의 시각적 차이가 약함
- **해결:** Before/Bad는 opacity 낮추기, After/Good은 accent + 약간의 배경색

### 4. 헤드라인이 모든 슬라이드에서 비슷한 크기
- S1의 title과 S6의 code 슬라이드가 같은 크기의 헤드라인 → 위계 구분 안 됨
- **해결:** 슬라이드 유형별 헤드라인 크기 스케일 (title > section > detail)

## 학습 포인트 — 규칙 추가 필요

### 신규 규칙 1: 슬라이드 유형별 헤드라인 스케일
```
title slide:   3.5vw (가장 큼)
section slide:  2.2vw (표준)
detail slide:   1.6vw (콘텐츠가 주인공인 슬라이드)
```

### 신규 규칙 2: 비교 슬라이드 시각적 무게 차별화
```
Before/Bad:  opacity 0.5, 텍스트 muted, 배경 없음 또는 연한 회색
After/Good:  opacity 1.0, 텍스트 primary, accent 컬러 요소 포함
무게 비율:   Before 30% / After 70% (동등하면 비교 효과 없음)
```

### 신규 규칙 3: 패턴 정체성 검증
```
레이아웃 패턴을 선택했으면, 그 패턴의 핵심 특성이 시각적으로 드러나야 한다.
- magazine-editorial → 비대칭 타이포 배치, 에디토리얼 여백
- quote-spotlight → 80%+ 여백, 인용문만 존재
- feature-callout → 주인공과 보조의 크기 차이 2배 이상
- alternating-zigzag → 좌→우, 우→좌 교차가 명확히 보여야 함
패턴 이름만 붙이고 실제로는 그냥 좌정렬이면 → 위반
```

### 신규 규칙 4: 콘텐츠 영역 상한선 (스타일별)
```
Exaggerated Minimalism: 콘텐츠 ≤ 35% of slide area
Standard Minimalism:    콘텐츠 ≤ 50%
Neobrutalism/Memphis:   콘텐츠 ≤ 60%
Bento/Dashboard:        콘텐츠 ≤ 75%
```

## 종합 점수 자가 평가

| 항목 | 점수 (100) | 이유 |
|------|:---------:|------|
| 그리드 정합성 | 60 | 12컬럼 기반이지만 수직 정렬 불일관 |
| 시각 계층 | 50 | 헤드라인 크기가 슬라이드 유형 무관하게 동일 |
| 스타일 정체성 | 45 | "Exaggerated" 느낌 부족. 여백이 극단적이지 않음 |
| 레이아웃 변주 | 65 | 7패턴 모두 다르지만, 패턴 정체성이 약함 |
| 비교 표현 | 35 | Before/After 무게 차이 거의 없음 |
| 타이포 규칙 | 70 | readable/decorative 구분은 됨 |
| 전체 인상 | 50 | 미니멀하긴 한데 "exaggerated"는 아님 |
| **종합** | **53** | 이전 대비 개선됐지만 "감각적"이라 하기엔 부족 |
