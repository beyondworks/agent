# 채널 기획서: 오리의 AI 인사이트

## 컨셉

캐릭터: 개발자 오리 (플랫 일러스트, 흰색 몸체, 주황 부리, 헤드셋 착용, 코드 심볼)
- 참고 이미지: `Image/Frame 5.png` (헤드셋+코딩), `Image/Frame 6.png` (커피)

채널명: **애덕이** (AI덕 / AI덕후의 줄임말 + 오리=덕)

타겟: 비개발자, 일반인 (20~40대). "AI가 뭔데? 나도 쓸 수 있어?"
톤: 유머 + 정보. 오리가 웃기면서도 핵심을 짚어주는 캐릭터.
형식: 숏폼 위주 (30~60초), 반응 좋은 소재는 롱폼(3~5분)으로 확장

## 캐릭터 설정

이름: 애덕이 (AIDuck)
직업: 실리콘밸리 출신 시니어 개발자... 였는데 퇴사하고 유튜버
성격:
- 복잡한 걸 쉽게 설명하는 재능
- 커피 중독 (5잔/일)
- 가끔 코드 얘기하다가 흥분하면 날개 퍼덕
- "덕!" = "진짜?" (감탄사)
- 전문용어 쓰다가 본인이 멈추고 "아 쉽게 말하면..." 하는 패턴

말투:
- 반말 + 친근체 ("~거든", "~잖아", "~인 거지")
- 전문용어 → 즉시 일상 비유 전환
- 예: "트랜스포머 아키텍처... 아 쉽게 말하면, 시험 볼 때 중요한 부분에 형광펜 치잖아? 그거야 덕."

## 콘텐츠 카테고리 (4개 시리즈)

### 시리즈 1: "덕설명" (메인, 주 3회)
AI 개념/뉴스를 30~45초로 쉽게 설명
- "ChatGPT가 거짓말하는 이유"
- "AI가 그림 그리는 원리 30초 설명"
- "Sora가 영상 만드는 방법"
- "AI 면접관이 나를 평가하는 기준"

### 시리즈 2: "덕실험" (주 1회)
AI 도구를 직접 써보고 결과를 보여줌
- "AI한테 이력서 써달라고 했더니"
- "AI로 퇴사 이메일 쓰기"
- "AI가 추천한 주식 1달 후 결과"
- "AI한테 소개팅 대화법 물어봄"

### 시리즈 3: "덕뉴스" (주 2회)
그 주의 AI 뉴스 핵심 3줄 요약
- "이번 주 AI 3줄 요약"
- "구글이 또 뭘 만들었는데..."
- "일론 머스크가 이번엔..."

### 시리즈 4: "덕 vs AI" (격주 1회, 롱폼 확장용)
오리가 AI와 대결하는 포맷 (3~5분)
- "오리 vs ChatGPT: 누가 더 웃긴 개그를 쓰나"
- "오리 vs AI: 라면 레시피 대결"
- "오리 vs AI: 연애 상담 누가 잘하나"

## 제작 파이프라인

### 숏폼 (30~60초)
1. 소재 선정: 최신 AI 뉴스/트렌드에서 일반인이 궁금할 만한 주제
2. 스크립트: 도입(3초, 후킹) → 설명(20~40초) → 마무리(5초, 애덕이 한마디)
3. 비주얼: 오리 캐릭터 일러스트 3~5장 (표정/포즈 변화) + 모션그래픽
4. 오디오: TTS 나레이션 (오리 말투) + BGM + 효과음
5. 자막: 핵심 키워드 강조 자막 필수

### 제작 도구 (기존 auto-video 파이프라인 활용)
- 스크립트: Gemini API (scriptwriter 에이전트)
- 이미지: 오리 캐릭터 기본 포즈 세트 (사전 제작) + Imagen 4 (배경/보조 그래픽)
- 나레이션: ElevenLabs TTS
- 합성: MoviePy (compositor.py) — Ken Burns + 자막 오버레이
- 메타데이터: youtube_metadata.py

## 숏폼 스크립트 샘플 5개

### 샘플 1: "ChatGPT가 거짓말하는 이유"
```
[장면 1 - 오리가 놀란 표정] (3초)
자막: "ChatGPT가 거짓말을 한다고?"
나레이션: "야, 너 ChatGPT가 가끔 거짓말하는 거 알아?"

[장면 2 - 오리가 칠판 앞에서 설명] (15초)
자막: "할루시네이션 = AI의 뇌피셜"
나레이션: "이걸 '할루시네이션'이라고 하는데... 아 쉽게 말하면, 시험 볼 때 모르는 문제 나오면 아는 척 적잖아? 그거야 덕. AI도 모르면 그럴듯하게 지어내거든."

[장면 3 - 오리가 자신감 있는 포즈] (10초)
자막: "확인하는 법"
나레이션: "그래서 AI가 말한 거 그냥 믿으면 안 돼. 중요한 건 꼭 검색해서 확인해봐. 특히 숫자나 날짜는 거의 틀려."

[장면 4 - 오리가 커피 마시며] (5초)
자막: "덕!"
나레이션: "AI도 완벽하지 않은 거지, 덕."
```

### 샘플 2: "AI가 내 일자리 뺏을까?"
```
[장면 1 - 오리가 걱정하는 표정] (3초)
자막: "AI 때문에 실직?"
나레이션: "AI가 내 직업 뺏어간다고? 솔직히 말할게."

[장면 2 - 오리가 그래프 가리키며] (15초)
자막: "대체 X → 도구 O"
나레이션: "AI가 사람을 대체하는 게 아니라, AI 쓸 줄 아는 사람이 못 쓰는 사람을 대체하는 거거든. 포토샵 나왔을 때 화가 다 사라졌어? 아니잖아."

[장면 3 - 오리가 키보드 치며] (10초)
자막: "AI는 도구, 쓰는 건 너"
나레이션: "중요한 건 AI를 얼마나 잘 활용하느냐야. 그러니까 무서워하지 말고 일단 써봐."

[장면 4 - 오리 윙크] (5초)
자막: "덕!"
나레이션: "나도 AI로 이 영상 만들었거든, 덕."
```

### 샘플 3: "프롬프트 엔지니어링이 뭔데"
```
[장면 1 - 오리 갸우뚱] (3초)
자막: "프롬프트 엔지니어링?"
나레이션: "프롬프트 엔지니어링... 이름은 거창한데."

[장면 2 - 오리가 식당에서 주문하는 장면] (20초)
자막: "AI한테 잘 시키는 법"
나레이션: "쉽게 말하면, 식당에서 주문 잘하는 거야. '맛있는 거 주세요' 하면 뭐가 나올지 모르잖아? 근데 '매운 거 빼고, 밥 많이, 국물 따로' 이러면 원하는 게 나오거든. AI도 똑같아."

[장면 3 - 오리가 예시 보여줌] (10초)
자막: "구체적으로 = 결과물 좋게"
나레이션: "구체적으로 시키면 구체적으로 나와. 역할, 형식, 예시 이 세 개만 넣어봐."

[장면 4 - 오리 엄지척] (5초)
자막: "덕!"
나레이션: "이것만 알면 너도 AI 고수, 덕."
```

### 샘플 4: "AI가 그림 그리는 원리"
```
[장면 1 - 오리가 그림 보며 감탄] (3초)
자막: "이걸 AI가 그렸다고?"
나레이션: "이 그림, 사람이 안 그렸어. AI가 그렸거든."

[장면 2 - 오리가 TV 노이즈 화면 가리킴] (15초)
자막: "노이즈 → 그림"
나레이션: "원리가 신기한데, TV 지직거리는 거 있잖아? 거기서 시작해서 노이즈를 조금씩 지워가는 거야. 마치 대리석 조각가가 돌 깎아서 형태 만드는 것처럼."

[장면 3 - 오리가 텍스트 입력하는 장면] (10초)
자막: "글자 → 이미지"
나레이션: "네가 '바다 위 오리' 라고 치면, AI가 '바다', '위', '오리'를 이해하고 그에 맞게 노이즈를 깎아내는 거지."

[장면 4 - 오리 셀카 포즈] (5초)
자막: "덕!"
나레이션: "근데 내 얼굴은 왜 매번 다르게 그려... 덕."
```

### 샘플 5: "AI 번역기가 파파고보다 나은 이유"
```
[장면 1 - 오리가 외국어에 당황] (3초)
자막: "번역기 쓰는데 이상해"
나레이션: "아직도 구글 번역기 쓰면서 뭔가 이상하다고 느끼지?"

[장면 2 - 오리가 비교하는 장면] (15초)
자막: "단어 번역 vs 맥락 번역"
나레이션: "예전 번역기는 단어 하나하나 바꾼 거야. 근데 요즘 AI 번역은 문장 전체를 이해해. 'I'm blue'를 '나는 파란색'이 아니라 '나 우울해'로 이해하는 거지."

[장면 3 - 오리가 추천하는 장면] (10초)
자막: "DeepL, ChatGPT 번역 추천"
나레이션: "그래서 요즘은 DeepL이나 ChatGPT한테 번역 시키면 진짜 자연스러워. 특히 긴 문장일수록 차이가 커."

[장면 4 - 오리가 영어로 말하다 포기] (5초)
자막: "덕!"
나레이션: "나도 영어는 AI한테 맡겨... 덕."
```

## 성장 로드맵

### 1개월차: 테스트 (숏폼 20~30개)
- "덕설명" 위주로 매일 1개
- 썸네일/후킹 A/B 테스트
- 반응 좋은 소재 패턴 파악

### 2~3개월차: 시리즈 확립 (주 5회)
- 덕설명 3회 + 덕뉴스 2회
- 1000 구독자 달성 → 커뮤니티 탭 활용
- 덕실험 시작

### 4~6개월차: 롱폼 확장
- "덕 vs AI" 롱폼 시리즈 시작
- 숏폼 → 롱폼 유입 퍼널 구축
- 굿즈/스티커 검토

## 차별화 포인트
1. 캐릭터 IP — 사람 의존 없이 확장 가능 (굿즈, 이모티콘, 웹툰)
2. 개발자 출신 오리 — "진짜 아는 오리"라는 신뢰감
3. 유머 + 비유 — 기술 채널의 딱딱함 제거
4. 제작 자동화 — auto-video 파이프라인으로 대량 생산 가능
