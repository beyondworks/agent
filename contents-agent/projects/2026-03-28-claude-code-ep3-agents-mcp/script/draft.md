# Claude Code 완전정복 EP.3 — 에이전트와 MCP

## 메타 정보

- 총 글자 수: 약 3,500자 (공백 포함)
- 예상 러닝타임: 6분 50초
- 씬 수: 9개

## Scene 1: 후킹

- Duration: 15초
- Visual: 슬래시 커맨드 → 에이전트 스폰 애니메이션 (1→다수 분기)

지난 편에서 슬래시 커맨드를 다뤘습니다.

슬래시 커맨드는 사람이 하나하나 시키는 겁니다.

에이전트는 다릅니다. 일을 맡기면 알아서 판단하고, 알아서 끝냅니다.

오늘부터 진짜 자동화입니다.

## Scene 2: 에이전트가 뭔가

- Duration: 30초
- Visual: 사장 → 팀장 → 팀원 비유 일러스트 + 독립 컨텍스트 박스 → 결과만 반환

에이전트는 클로드가 부르는 또 다른 클로드입니다.

회사로 치면 이렇습니다. 내가 사장이고 클로드가 팀장입니다. 팀장한테 일을 시키면, 팀장이 알아서 팀원을 불러서 처리합니다. 그 팀원이 에이전트입니다.

팀원은 자기 방에서 독립적으로 일하고, 끝나면 결과 보고서만 올립니다. 중간에 어떤 삽질을 했는지는 팀장한테 안 보입니다.

제 관점에서 이게 왜 중요하냐면, 파일 백 개짜리 분석을 시켜도 메인 대화가 지저분해지지 않습니다. 일 시킨 사람은 결과만 받으면 됩니다.

## Scene 3: 내장 에이전트

- Duration: 60초
- Visual: 에이전트 카드 3장 (Explore/Plan/General) + 터미널 실행 화면

기본으로 들어 있는 에이전트가 세 종류입니다.

첫 번째, 익스플로어. 정찰병입니다. 코드 구조를 빠르게 훑고 와서 보고합니다. "로그인 로직이 어디에 있어?" 이런 질문에 쓰면 됩니다. 가볍고 빠른 모델로 돌아서 비용도 적습니다.

두 번째, 플랜. 설계사입니다. "이 기능 어떻게 만들면 좋을까?" 물어보면 코드를 읽고 계획을 세워줍니다. 대신 직접 코드를 고치지는 않습니다. 보기만 하니까 안전합니다.

세 번째, 제너럴 퍼포스. 만능 해결사입니다. 코드 읽기, 수정, 터미널 명령까지 전부 할 수 있습니다.

여기서 규칙이 하나 있습니다. 팀원은 또 다른 팀원을 부를 수 없습니다. 팀원이 팀원을 부르고, 그 팀원이 또 팀원을 부르면 끝이 없으니까요. 팀원을 부를 수 있는 건 팀장뿐입니다.

## Scene 4: 커스텀 에이전트

- Duration: 60초
- Visual: .claude/agents/ 폴더 구조 + 마크다운 파일 내용 + 스폰 데모

여기까지는 기본입니다. 진짜 강력한 건 내가 원하는 전문가를 직접 만드는 겁니다.

방법이 놀라울 정도로 쉽습니다. 폴더에 파일 하나 넣으면 끝입니다.

예를 들어 "보안 전문가 점 엠디"라는 파일을 만듭니다. 안에 이름, 설명, 쓸 수 있는 도구를 적고, 아래에 역할을 써줍니다. "당신은 시니어 보안 엔지니어입니다. 코드에 보안 구멍이 없는지 검토하세요."

이 파일 하나로 보안 전문가가 팀에 합류합니다.

제 관점에서 이건 채용과 같습니다. 이력서를 한 장 쓰면 그 역할의 전문가가 바로 투입됩니다. 해고도, 면접도 없습니다.

작업 공간도 분리됩니다. 에이전트마다 독립된 작업 공간을 줄 수 있어서, 다섯 명이 같은 프로젝트에서 동시에 일해도 서로 부딪히지 않습니다.

## Scene 5: MCP란

- Duration: 35초
- Visual: USB 포트 비유 → 만능 어댑터 → 이천 개 서버 숫자 카운트업

여기까지가 에이전트입니다. 이제 엠시피.

엠시피는 에이아이와 외부 서비스를 연결하는 규격입니다.

비유하면 만능 어댑터입니다. 깃허브가 이 규격에 맞는 플러그를 하나 만들면, 클로드코드에서도 쓰고, 커서에서도 쓰고, 브이에스코드에서도 씁니다. 하나 만들면 어디서든 꽂아서 쓰는 겁니다.

노션, 슬랙, 스트라이프, 수파베이스. 지금 이 규격에 맞는 플러그가 이천 개 넘게 나와 있습니다. 오픈에이아이, 구글, 마이크로소프트도 이 규격을 쓰고 있습니다.

## Scene 6: MCP 실전

- Duration: 70초
- Visual: 터미널에서 claude mcp add 실행 + 각 서버별 기능 카드

연결은 한 줄이면 됩니다.

터미널에 "클로드 엠시피 애드" 치고, 연결할 서비스 이름이랑 주소를 넣으면 끝입니다.

연결하면 뭘 할 수 있냐.

깃허브를 연결하면, 클로드가 직접 코드 변경 요청을 만들고, 이슈를 관리합니다. 깃허브 웹사이트에 들어갈 필요가 없습니다.

노션을 연결하면, 페이지를 검색하고, 데이터를 조회하고, 문서를 자동으로 만들어 줍니다.

슬랙을 연결하면, 특정 채널의 대화를 읽고, 메시지를 보내고, 대화 흐름을 따라갑니다.

수파베이스를 연결하면, 데이터베이스에 직접 질문하고, 구조를 바꾸고, 서버 코드를 배포합니다.

이게 전부 하나의 대화 안에서 됩니다. 탭을 전환하거나 창을 바꿀 필요가 없습니다.

## Scene 7: 에이전트 더하기 MCP

- Duration: 60초
- Visual: 파이프라인 플로우 (Slack → Claude → GitHub → Slack) + Agent Teams 분할 화면

에이전트와 엠시피를 합치면 이런 시나리오가 가능합니다.

슬랙에 누군가 버그를 보고합니다. 클로드가 그 메시지를 읽습니다. 코드를 분석해서 원인을 찾습니다. 고칩니다. 깃허브에 수정 요청을 올립니다. 슬랙에 "고쳤습니다" 알림을 보냅니다.

이 과정에서 사람이 한 일은 없습니다.

여기서 더 나아가면 에이전트 팀입니다. 보안 전문가, 성능 분석가, 테스트 담당자를 동시에 띄워서 하나의 수정 사항을 세 관점으로 동시에 검토합니다. 팀원끼리 의견도 주고받습니다.

제 관점에서 이건 개발 팀을 통째로 자동화하는 겁니다. 기획서를 쓰면 알아서 개발하고, 검토하고, 배포하는 흐름이 여기서 시작됩니다.

## Scene 8: 주의할 점

- Duration: 30초
- Visual: 경고 아이콘 + 팁 카드 3장

세 가지만 기억하세요.

첫째, 플러그를 많이 꽂으면 클로드가 기억할 수 있는 양이 줄어듭니다. 필요한 것만 연결하세요. 세 개만 동시에 켜도 전체 기억 용량의 칠십 퍼센트를 잡아먹을 수 있습니다.

둘째, 에이전트 팀 기능은 아직 실험 단계입니다. 중요한 업무에 바로 쓰기보다 테스트용으로 먼저 써보세요.

셋째, 외부 플러그는 앤쓰로픽이 직접 검증한 게 아닙니다. 데이터베이스를 연결할 때는 읽기만 가능한 계정으로 연결하세요.

## Scene 9: 정리와 행동 제시

- Duration: 30초
- Visual: 실행 체크리스트 카드 + EP.4 예고

정리합니다.

오늘 당장 해볼 세 가지.

하나, 커스텀 에이전트 파일을 하나 만들어 보세요. 내가 원하는 전문가를 채용하는 겁니다.

둘, 엠시피로 깃허브나 노션을 연결해 보세요. 플러그 하나 꽂는 거라 일 분이면 됩니다.

셋, 에이전트한테 연결된 서비스로 뭔가를 시켜 보세요. 그 순간 코딩 도구가 아니라 자동화 시스템이 됩니다.

다음 편에서는 클로드코드 실전 세팅을 다룹니다. 클로드엠디 작성법부터 에이전트 조합 패턴까지, 제대로 쓰기 위한 초기 세팅 전부 알려드립니다.

구독과 알림 설정 부탁드립니다.
