# 2026년, AI 에이전트는 왜 90%가 실패할까
- 목표 러닝타임: 3분
- 총 글자 수: 1,543자
- 씬 수: 7개

---

## Scene 1: 후킹 - 역설적 선언
### Duration: 13초
### Narration
올해 기업 앱의 40%가 AI 에이전트를 품을 거라고 합니다. [sources.md L16] 그런데 동시에, AI 에이전트 프로젝트의 80에서 90%는 실패합니다. [sources.md L18] 이 두 숫자가 동시에 존재하는 상황. 뭔가 이상하지 않습니까.
### Visual Direction
#001C2F 배경 위에 "40%" 숫자가 #06E5AC 컬러로 좌측에서 타이핑되듯 등장. 0.5초 후 우측에서 "90%" 숫자가 #FF4D4D 컬러로 슬라이드 인. 두 숫자 사이에 "vs" 텍스트가 페이드인되며 긴장감 조성. 하단에 CLI 커서가 깜빡이는 모션.

## Scene 2: 에이전트가 뭔지부터
### Duration: 25초
### Narration
에이전트를 한마디로 하면, 스스로 판단하고 행동하는 AI입니다. 챗지피티한테 "이거 알려줘"라고 묻는 건 챗봇이고, "이 업무를 처음부터 끝까지 처리해줘"라고 맡기는 게 에이전트입니다. 비유하자면 이렇습니다. 챗봇은 콜센터 상담원이에요. 질문하면 답해줍니다. 에이전트는 신입 직원입니다. 업무를 주면 알아서 단계를 나누고, 필요한 도구를 찾아 쓰고, 결과를 가져옵니다.
### Visual Direction
화면 좌측에 말풍선 아이콘(챗봇)이 MUI 카드 스타일로 등장, 우측에 사람 아이콘+도구 아이콘(에이전트)이 순차적으로 슬라이드 인. 챗봇 카드에는 "Q&A" 라벨, 에이전트 카드에는 "판단 > 실행 > 보고" 플로우 다이어그램이 SVG 라인 드로잉 애니메이션으로 그려짐. 비유 전환 시 콜센터 헤드셋 SVG 아이콘이 페이드인 후 신입 직원 데스크 아이콘으로 모프 전환.

## Scene 3: 왜 지금 폭발하는가
### Duration: 28초
### Narration
그런데 에이전트라는 개념은 새로운 게 아닙니다. 왜 하필 2026년에 폭발하느냐. 세 가지가 동시에 맞아떨어졌기 때문입니다. 첫째, 모델 성능이 임계점을 넘었습니다. 둘째, 엠씨피 같은 표준 프로토콜이 생겼습니다. 파이어쉽이 이걸 "AI의 USB-C"라고 비유했는데, [sources.md L14] 충전기마다 단자가 다르던 시절에서 하나로 통일된 것과 같습니다. 셋째, 멀티 에이전트 시스템, 그러니까 에이전트끼리 협업하는 구조에 대한 기업 문의가 1,445% 급증했습니다. [sources.md L16]
### Visual Direction
"2026" 연도 텍스트가 #06E5AC로 화면 중앙에 큰 글씨로 타이핑. 이후 3개의 MUI 카드가 1-2-3 순서로 아래에서 위로 스택되며 등장. 각 카드에 핵심 키워드: (1) "모델 성능" + 상승 그래프 SVG (2) "표준 프로토콜" + USB-C 커넥터 SVG 아이콘이 여러 단자에서 하나로 합쳐지는 모션 (3) "1,445%" 숫자가 카운트업 애니메이션으로 표시. 카드 간 전환은 0.3초 슬라이드.

## Scene 4: 에이전트 시장의 현실
### Duration: 27초
### Narration
여기서 냉정하게 볼 게 있습니다. 레딧 커뮤니티에서 지금 가장 많이 나오는 비판이 뭐냐면, "에이전트라고 마케팅하는 것들 대부분이 그냥 챗봇에 자동화를 씌운 것"이라는 겁니다. [sources.md L18] 제 경험상으로도 그렇습니다. 이커머스에서 "AI 에이전트 도입했다"는 프로젝트를 들여다보면, 챗봇이 주문 상태를 읽어주는 수준인 경우가 대부분이었습니다. 자동화 워크플로우에 AI 라벨을 붙인 거지, 스스로 판단하는 구조는 아닌 거예요.
### Visual Direction
CLI 터미널 창이 화면 중앙에 등장. "$ reddit search 'AI agent' --sort=top" 명령어가 타이핑되고, 결과로 비판 코멘트들이 스크롤업. 이후 터미널이 좌측으로 밀리며 우측에 비교 다이어그램 등장: "마케팅" 라벨 아래 [챗봇 + 자동화 = "에이전트"] vs "실제" 라벨 아래 [판단 + 도구 사용 + 자율 실행 = 에이전트]. 실제 쪽에 #06E5AC 보더 강조. 글래스모피즘 배경 처리.

## Scene 5: 기획자 렌즈 - 설계가 핵심인 이유
### Duration: 32초
### Narration
그래서 진짜 중요한 건 뭐냐. 에이전트를 쓰느냐 마느냐가 아니라, 어떤 구조로 쓰느냐입니다. 코딩 에이전트 시장만 봐도, 클로드코드가 레딧 멘션 226건으로 1위이고, 커서는 IDE 표준, 데빈은 가장 자율적인 에이전트로 자리 잡고 있습니다. [sources.md L22] 그런데 도구가 좋아도 브리프가 엉망이면 결과도 엉망입니다. 컨설팅 보고서에서 "그래서 뭘 하라는 건데?"가 빠지면 쓸모없는 것처럼, 에이전트한테 주는 지시에 목적과 조건과 판단 기준이 없으면 그냥 비싼 자동완성이 됩니다.
### Visual Direction
3개의 MUI 카드가 수평으로 나란히 등장: 클로드코드(멘션 226), 커서(IDE 표준), 데빈(자율 에이전트). 각 카드에 해당 로고 SVG와 포지셔닝 한 줄. 2초 후 카드들이 축소되며 상단으로 이동하고, 하단에 "브리프 구조" 다이어그램이 라인 드로잉으로 그려짐: [목적] -> [조건] -> [판단 기준] -> [결과]. 구조가 없는 경우 라인이 끊어지며 "자동완성" 라벨이 #FF4D4D로 표시.

## Scene 6: 오늘 당장 해볼 것
### Duration: 28초
### Narration
오늘 하나만 해보시죠. 지금 반복하고 있는 업무 하나를 골라서, 이 세 가지를 적어보세요. 이 일의 목적이 뭔지, 판단 기준이 뭔지, 끝난 상태가 어떤 건지. 이게 결국 브리프입니다. 브랜드 가이드라인이 있으면 누가 만들어도 톤이 맞듯이, AI한테도 이 구조를 주면 일관된 결과가 나옵니다. 기획서를 넘기고 2주를 기다리던 시절이 있었습니다. 지금은 이 구조를 주면 30분이면 결과를 봅니다. 도구는 그 다음입니다. 구조가 먼저예요.
### Visual Direction
화면에 MUI 다크모드 메모장 UI가 등장. 빈 상태에서 타이핑 애니메이션으로 세 줄이 작성됨: "1. 목적:", "2. 판단 기준:", "3. 완료 상태:". 각 줄 작성 시 #06E5AC 커서 깜빡임. 작성 완료 후 메모장이 축소되며 좌측으로 이동, 우측에 타임라인 SVG 등장: "2주" 구간이 압축되며 "30분"으로 줄어드는 모션. 하단에 "구조 > 도구" 텍스트가 #06E5AC로 페이드인.

## Scene 7: 마무리
### Duration: 27초
### Narration
정리하겠습니다. 2026년, AI 에이전트는 진짜 폭발합니다. 그런데 90%는 실패할 겁니다. 차이를 만드는 건 도구가 아니라 설계입니다. 처음 AI로 만든 건 보기엔 그럴듯했는데, 쓸 수가 없었어요. 기획 없이 시작한 거였거든요. 그때 배운 겁니다. 코드를 모르니까 오히려 구조에 집중하게 됩니다. 나무가 아니라 숲을 먼저 보는 거예요. 다음 영상에서는 실제로 에이전트를 설계하는 과정을 단계별로 보여드리겠습니다. 구독해두시면 놓치지 않습니다.
### Visual Direction
앞선 씬의 핵심 키워드들("40% vs 90%", "구조", "브리프", "목적-기준-상태")이 플로팅 카드로 화면에 흩어져 있다가 중앙으로 모이며 하나의 문장으로 합쳐짐: "설계가 먼저다". #06E5AC 글로우 이펙트. 2초 유지 후 채널 로고가 우하단에서 슬라이드 인. 구독 버튼 SVG 아이콘이 #06E5AC 컬러로 펄스 애니메이션. CLI 스타일로 "$ subscribe --channel" 텍스트 타이핑 모션.
