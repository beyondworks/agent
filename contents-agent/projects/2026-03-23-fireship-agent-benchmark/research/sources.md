## Fireship 포맷 분석

> Fireship의 에이전트 관련 영상에서 발견되는 공통 포맷 특성을 분석.

- **영상 길이**: 이중 구조. "100 Seconds of Code"는 2분 내외 (중독성 루프 유도), "The Code Report"는 8-12분 (트렌드 딥다이브). 에이전트 관련 영상은 주로 Code Report 형식(8-12분).

- **후킹 방식**: 자극적/도발적 제목으로 클릭 유도. "AI just officially took our jobs", "I gave Claude root access to my server", "The vibe coding mind virus" 등. 첫 5-10초에 충격적 진술이나 위트 있는 자조적 발언으로 시작. 프로그래머라면 공감할 "내부자 농담"을 초반에 배치.

- **설명 구조**: (1) 충격적 후킹 (2) 개념의 핵심을 일상적 비유로 축약 (MCP = "AI의 USB-C") (3) 기술적 깊이를 빠른 속도로 전달 (4) 실제 코드/데모 시연 (5) 위험성/한계 경고로 균형 잡기 (6) 짧은 마무리. 한 방향으로만 주장하지 않고 양면을 보여주는 것이 특징.

- **비주얼 스타일**: 정지 이미지와 모션 그래픽 조합. Figma/After Effects로 제작된 그래픽 자산. 어두운 배경(다크 테마) 위에 선명한 코드 하이라이트. 밈과 영상 클립을 빠르게 삽입하여 시각적 재미 강화.

- **전환 속도**: 매우 빠름. 한 장면이 3-5초를 넘기지 않는 빠른 컷 편집. 고에너지 전환으로 시청자의 이탈을 방지. 복잡한 설명 시에만 잠시 속도를 늦춤.

- **유머/위트 활용**: 프로그래밍 문화를 공유하는 "인사이더 조크"가 핵심. 자조적이고 냉소적인 톤("As a programmer, I hate Devin. But as a YouTuber, Devin has been good to me"). 밈을 정보 전달의 보조 수단으로 활용. 과장법으로 관심을 끌되, 실제 기술 내용은 정확하게 전달.
