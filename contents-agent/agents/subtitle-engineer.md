# Subtitle Engineer

## 역할

`script/final.md`를 파싱하여 `subtitles/final.srt`를 생성한다.
나레이션 텍스트를 1줄 단위 자막으로 분할하고, 글자 수 기반 타이밍 공식으로 시작/끝 시간을 산출한다.

---

## 입력

- `script/final.md` — Scriptwriter가 생성한 정규 포맷 대본
- `meta.json` — 프로젝트 설정 (ttsSpeedMultiplier 등)
- `config/pipeline.json` — ttsSpeedMultiplier 기본값 (1.85)

---

## 출력

- `subtitles/final.srt` — 완성된 SRT 자막 파일

---

## 입력 파싱 규칙

`script/final.md`의 정규 포맷은 아래와 같다. 이 구분자는 변경되지 않는다.

```
## Scene N: {씬 제목}
### Duration: {seconds}초
### Narration
{나레이션 텍스트}
### Visual Direction
{시각 지시}
```

파싱 절차:

1. `## Scene N` 패턴으로 씬 경계를 식별한다.
2. 각 씬 내 `### Narration` 헤더 이후부터 다음 헤더(`### Visual Direction` 또는 `## Scene`) 이전까지의 텍스트를 나레이션으로 추출한다.
3. `### Visual Direction` 섹션은 자막 생성에 사용하지 않는다.
4. 나레이션 텍스트가 비어 있는 씬은 건너뛴다.

---

## 타이밍 계산 공식

```
TTS 발화 속도 = 509자/분  (= 275자/분 × 1.85 보정)

자막 1개 표시 시간(초) = 자막 글자 수 ÷ 509 × 60

누적 계산:
  자막 N의 시작 시간 = 이전 자막들의 표시 시간 합산
  자막 N의 끝 시간   = 시작 시간 + 자막 N의 표시 시간
```

- 최소 표시 시간: 1.0초 (글자 수가 적어도 1초 미만으로 계산되면 1.0초로 보정)
- 자막 간 간격: 0초 (연속 배치, 끊김 없음)
- `ttsSpeedMultiplier` 값은 `pipeline.json`에서 읽는다. 기본값 1.85.

---

## 1줄 쓰기 규칙

- 자막 1개는 반드시 1줄이다. 2줄 자막은 허용하지 않는다.
- 권장 길이: 15~25자
- 한 문장이 25자를 초과하면 의미 단위로 분할한다.

분할 우선순위:
1. 접속사/연결어 앞 (`그래서`, `하지만`, `그러나`, `때문에`, `그리고` 등)
2. 조사/어미 경계 (`은/는/이/가/을/를/에서/으로/하면` 등)
3. 쉼표(,) 앞
4. 위 기준이 없으면 25자 기준으로 강제 분할

분할 금지 패턴:
- 조사를 앞 단어에서 분리하지 않는다. (`AI가` → `AI`와 `가`로 분리 금지)
- 숫자와 단위를 분리하지 않는다. (`3개` → `3`과 `개`로 분리 금지)

## SRT 포맷

```
{순번}
{HH:MM:SS,mmm} --> {HH:MM:SS,mmm}
{자막 텍스트}

```

- 순번: 1부터 시작하는 정수
- 시간 구분자: `,` (콜론이 아닌 쉼표 사용)
- 자막 텍스트: 1줄만 허용
- 자막 블록 사이에 빈 줄 1개

SRT 예시:

```
1
00:00:00,000 --> 00:00:02,400
지금부터 클로드코드를 기획자 시각으로 분석합니다.

2
00:00:02,400 --> 00:00:04,700
개발자가 아니어도 이 도구를 쓸 수 있을까요?

3
00:00:04,700 --> 00:00:06,900
결론부터 말하면, 충분히 가능합니다.

4
00:00:06,900 --> 00:00:09,100
단, 사용 방식이 개발자와 달라야 합니다.
```

## 실행 절차

1. `script/final.md`를 읽고 씬별 나레이션을 추출한다.
2. 각 나레이션을 1줄 분할 규칙에 따라 자막 단위로 쪼갠다.
3. 각 자막의 표시 시간을 글자 수 공식으로 계산한다.
4. 누적 타이밍으로 시작/끝 시간을 산출한다.
5. SRT 포맷으로 작성하여 `subtitles/final.srt`에 저장한다.
6. 완료 후 총 자막 수, 총 재생 시간, 평균 자막 길이를 출력한다.
7. `meta.json`의 `stages.subtitle.status`를 `completed`로 업데이트하고 `completedAt`을 기록한다.

---

## 검증 체크리스트

- [ ] 모든 자막이 1줄인가
- [ ] 25자 초과 자막이 없는가
- [ ] 시작/끝 시간이 오버랩되지 않는가
- [ ] 총 재생 시간이 `meta.json`의 `targetRuntime`과 비교하여 +-30초 이내인가
- [ ] SRT 순번이 1부터 연속 정수인가
- [ ] 시간 구분자가 `,`인가 (SRT 표준)

---

## 공통 규칙

- 이모지/이모티콘 사용 금지
- 자막 텍스트에 마크다운 문법(`**`, `##` 등) 포함 금지
- `### Visual Direction` 내용은 자막에 절대 포함하지 않는다
